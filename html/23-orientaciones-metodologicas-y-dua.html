Enfoque metodológico del proyecto

La unidad didáctica dedicada al lenguaje procedimental en bases de datos se desarrolla mediante una metodología activa orientada al aprendizaje práctico y a la resolución de problemas técnicos reales.

El alumnado no se limita a reproducir ejercicios, sino que trabaja con herramientas profesionales y situaciones cercanas al entorno laboral del desarrollo de software, aplicando progresivamente procedimientos almacenados, funciones y triggers para automatizar operaciones dentro de MySQL.

La propuesta combina explicación guiada, práctica individual, resolución colaborativa de problemas y desarrollo de un pequeño proyecto final que integra todos los aprendizajes adquiridos.

Principios metodológicos que guían la unidad

Aprendizaje activo

El alumnado construye su aprendizaje mediante prácticas reales sobre bases de datos, evitando un enfoque exclusivamente teórico.

Progresión técnica

Las tareas están organizadas de forma gradual, comenzando con scripts básicos y avanzando hacia soluciones más complejas.

Aprendizaje aplicado

Los contenidos se relacionan con situaciones reales del ámbito del desarrollo de software y la administración de bases de datos.

Uso de herramientas profesionales

El alumnado utiliza entornos técnicos reales como MySQL, Docker o editores de código.

Metodología en el aula

Durante el desarrollo de la unidad se alternan diferentes tipos de actividades que permiten combinar explicación, práctica y reflexión.

  • Explicaciones breves del docente para introducir conceptos clave.
  • Demostraciones técnicas sobre el funcionamiento de MySQL.
  • Prácticas guiadas para consolidar el uso del lenguaje procedimental.
  • Resolución de problemas reales relacionados con la automatización de tareas.
  • Mini proyecto final que integra procedimientos, funciones y triggers.

Este enfoque permite que el alumnado aprenda haciendo, desarrollando tanto competencias técnicas como habilidades de análisis y documentación.

Atención a la diversidad y enfoque DUA

La propuesta se ha diseñado teniendo en cuenta los principios del Diseño Universal para el Aprendizaje (DUA), ofreciendo diferentes formas de acceso a los contenidos y de demostración del aprendizaje.

  • Materiales explicativos en diferentes formatos (texto, ejemplos, demostraciones).
  • Prácticas guiadas paso a paso para alumnado con mayor necesidad de apoyo.
  • Retos de ampliación para alumnado con mayor dominio técnico.
  • Trabajo colaborativo para favorecer el aprendizaje entre iguales.

Este enfoque permite adaptar la propuesta a distintos ritmos de aprendizaje sin modificar los objetivos de la unidad.

Herramientas digitales utilizadas en la metodología

Docker

Despliegue del entorno de trabajo con contenedores de MySQL y phpMyAdmin.

MySQL

Sistema gestor de bases de datos utilizado para desarrollar los scripts.

VSCode / AntiGravity

Edición de scripts SQL y apoyo con herramientas de IA.

GitHub

Organización y entrega del código desarrollado por el alumnado.

Recomendaciones para el profesorado

  • Dedicar tiempo inicial a comprobar que el entorno Docker funciona correctamente.
  • Ofrecer ejemplos de scripts completos antes de pedir al alumnado que los desarrolle.
  • Fomentar la resolución colaborativa de problemas técnicos.
  • Combinar el uso de herramientas de IA con actividades de comprobación del aprendizaje.
  • Solicitar pequeñas explicaciones orales del código para asegurar la comprensión.

Estas recomendaciones facilitan la aplicación práctica de la unidad y ayudan a evitar bloqueos técnicos durante las sesiones.
