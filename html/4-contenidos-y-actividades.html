4. Producto final

Reto final del proyecto

Al finalizar esta unidad, deberás desarrollar una solución funcional de automatización en MySQL aplicando procedimientos almacenados, funciones y/o triggers para resolver un caso práctico.

Este producto final integrará los aprendizajes trabajados durante las sesiones y servirá como evidencia principal de tu progreso técnico, tu capacidad de documentación y tu uso responsable de herramientas digitales.

Tu misión

Diseñar e implementar una propuesta técnica que automatice tareas dentro de una base de datos MySQL. Para ello, deberás analizar el caso planteado, seleccionar la solución más adecuada y desarrollar los scripts necesarios.

El objetivo no es únicamente que el código funcione, sino que esté bien estructurado, documentado, justificado y probado, mostrando una comprensión real de los contenidos de la unidad.

Qué debes entregar

1. Scripts SQL

Archivo o conjunto de scripts con procedimientos, funciones y/o triggers correctamente implementados.

2. Documentación

Breve explicación del problema planteado, la solución adoptada y el funcionamiento del código desarrollado.

3. Evidencias de prueba

Capturas, resultados o demostraciones que permitan comprobar que la solución funciona correctamente.

4. Entrega organizada

Publicación del código y de la documentación mediante las herramientas indicadas por el docente.

Características del producto final

  • Debe responder a una necesidad concreta de automatización.
  • Debe incluir al menos una solución procedimental desarrollada en MySQL.
  • Debe ser técnicamente correcta y coherente con el caso planteado.
  • Debe presentar código legible, ordenado y comentado.
  • Debe incorporar pruebas de funcionamiento.
  • Debe reflejar un uso adecuado de las herramientas digitales del proyecto.

Herramientas que podrás utilizar

  • MySQL y phpMyAdmin para crear y probar la solución.
  • Docker para desplegar el entorno de trabajo.
  • VSCode o AntiGravity para editar scripts SQL.
  • GitHub para organizar y entregar el código.
  • Google Drive para compartir la documentación.
  • OBS o Loom si se solicita una breve demostración grabada.

Cómo puedes organizar tu trabajo

Paso 1

Analiza el caso práctico y detecta qué tareas conviene automatizar.

Paso 2

Decide si necesitas procedimientos, funciones, triggers o una combinación de ellos.

Paso 3

Desarrolla los scripts y comprueba su funcionamiento con distintos casos de prueba.

Paso 4

Documenta la solución y organiza la entrega final de forma clara.

Qué se tendrá en cuenta en la evaluación

Se valorará la corrección técnica del código, la estructura de la solución, el uso adecuado del lenguaje procedimental, la documentación presentada y las pruebas realizadas para validar el funcionamiento.

Consejo importante

No se trata solo de “hacer que funcione”, sino de demostrar que entiendes por qué has elegido esa solución y cómo responde al problema planteado.

Meta final

Este producto final te permitirá aplicar los conocimientos adquiridos en un contexto técnico realista, integrando programación SQL, documentación, resolución de problemas y uso responsable de herramientas digitales.

El resultado debe mostrar que eres capaz de diseñar una solución útil, bien pensada y técnicamente sólida dentro del entorno de bases de datos trabajado en la unidad.
